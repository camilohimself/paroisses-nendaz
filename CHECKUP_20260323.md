## 🟠 Majeurs (4)

- **API /api/horaires sans validation d'input** — `app/api/horaires/route.ts:61-64`
  - Probleme : 3 `searchParams` (sector, calendarId, months) lus directement sans validation. Pas de zod, joi, ni schema. Un `calendarId` malveillant pourrait etre passe a l'API Google Calendar, et `months` est parse via `parseInt` sans borne.
  - Fix recommande : ajouter une validation minimale (zod ou manuelle) — whitelist des sector/calendarId autorises, borner months entre 1 et 12

- **ESLint ignore pendant le build** — `next.config.ts:4-7`
  - Probleme : `ignoreDuringBuilds: true` masque toutes les erreurs ESLint en production. Des bugs potentiels passent silencieusement.
  - Fix recommande : retirer `ignoreDuringBuilds: true`, corriger les erreurs ESLint residuelles

- **13 pages paroisses bloquees par robots.txt** — `public/robots.txt`
  - Probleme : le robots.txt en mode whitelist n'autorise que 7 paroisses (Aproz, Basse-Nendaz, Brignon, Fey, Haute-Nendaz, Saclentse, Veysonnaz) mais le site genere 20 slugs via `[slug]`. Les pages suivantes sont bloquees pour Google :
    - Chapelles : beuson, bleusy, clebes, condemines, planchouet, rairettes, st-barthelemy, st-michel-hn, st-sebastien
    - Pages speciales : equipe-pastorale, evenements-paroissiaux, foyer-ma-vallee, ems-vergers
  - Fix recommande : ajouter les `Allow:` manquants dans robots.txt pour les pages a valeur SEO locale (ex: "eglise beuson nendaz")

- **Multiple H1 par page** — `app/avent/page.tsx`, `app/careme2026/page.tsx`
  - Probleme : avent/page.tsx contient 3 `<h1>`, careme2026/page.tsx contient 3 `<h1>`, careme2026/palettes contient 2 `<h1>`. Un seul H1 par page pour le SEO.
  - Fix recommande : garder un seul `<h1>` principal, convertir les autres en `<h2>`

## 🟡 Mineurs (8)

- **Pas de security headers** — `next.config.ts`
  - Aucun header de securite configure (X-Frame-Options, Strict-Transport-Security, X-Content-Type-Options, Referrer-Policy). Recommande pour un site en production sur Vercel.

- **alt="" sur images VideoHero** — `components/VideoHero.tsx:92,111`
  - 2 images avec `alt=""`. Si decoratives → OK. Si significatives → ajouter un texte alt descriptif.

- **public/ pese 55MB** — `public/`
  - Videos hero (attendu), PDFs documents (attendu), mais `rairettes/hero-desktop.jpg` et `.webp` >500KB. Verifier si les versions WebP suffisent et supprimer les JPG redondants.

- **Tailwind v4 avec config v3** — `tailwind.config.ts`
  - Le projet utilise Tailwind v4 (`^4.1.12`) mais garde un `tailwind.config.ts` style v3 avec `content[]` manuel. En TW v4, la detection est automatique. Fonctionne mais legacy.

- **14 fichiers >200 lignes, 8 >800 lignes**
  - `saints-enrichis-data.ts` (1144 — donnees, acceptable)
  - Pages avent/mission-* (840-1066 lignes) et parcours-pelerins/* (830-910 lignes) : contenu riche mais monolithique
  - `careme2026/page.tsx` (1003 lignes) : composant le plus complexe

- **1 TODO en attente** — `app/admin/dashboard/page.tsx:2`
  - `// TODO: Re-enable auth when database is configured`

- **1 type `any`** — `app/paroisses/[slug]/page.tsx:95`
  - `formatHoraires = (horaires: any)` — typer proprement l'objet horaires

- **Prisma sans migrations versionnees** — `prisma/`
  - Aucune migration Prisma trouvee. La DB est geree via `db push` (pas de controle de version des schemas). Acceptable pour SQLite dev, mais risque en production si la structure evolue.
